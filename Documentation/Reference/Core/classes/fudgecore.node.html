addEventListener

  • addEventListener(_type: EVENT | string, _handler: EventListener, _capture?: boolean): void

  • Overrides EventTargetƒ.addEventListener

    • Defined in Node/Node.ts:320

    Adds an event listener to the node. The given handler will be called when a matching event is passed to the node. Deviating from the standard EventTarget, here the _handler must be a function and _capture is the only option.

    Parameters

    • _type: EVENT | string

      The type of the event, should be an enumerated value of NODE_EVENT, can be any string

    • _handler: EventListener

      The function to call when the event reaches this node

    • Default value _capture: boolean = false

      When true, the listener listens in the capture phase, when the event travels deeper into the hierarchy of nodes.

    Returns void

dispatchEvent

  • dispatchEvent(_event: Event): boolean

  • Overrides EventTargetƒ.dispatchEvent

    • Defined in Node/Node.ts:338

    Dispatches a synthetic event event to target. This implementation always returns true (standard: return true only if either event's cancelable attribute value is false or its preventDefault() method was not invoked) The event travels into the hierarchy to this node dispatching the event, invoking matching handlers of the nodes ancestors listening to the capture phase, than the matching handler of the target node in the target phase, and back out of the hierarchy in the bubbling phase, invoking appropriate handlers of the anvestors

    Parameters

    • _event: Event

      The event to dispatch

    Returns boolean
